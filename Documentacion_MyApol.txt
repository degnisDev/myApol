=========================================================
  RESUMEN TÉCNICO DEL PROYECTO MYAPOL - DEGNISDEV (2026)
=========================================================

1. ARQUITECTURA GENERAL
   - Frontend: HTML5, CSS3, JavaScript (Vanilla), Bootstrap 5, Bootstrap Icons.
   - Backend: Python 3, Flask (RESTful API).
   - Base de Datos: SQLite (database.db).
   - Patrón de Diseño: Cliente-Servidor (API REST), SPA parcial (Single Page Application UX).

2. CARACTERÍSTICAS PRINCIPALES IMPLEMENTADAS
   - Diseño Premium "Glassmorphism" con tema oscuro inspirado en Apple.
   - Sistema de Routing Dinámico (frontend consumiendo backend vía Fetch API).
   - Barra de Navegación con detección de sesión mediante `localStorage`.
   - Búsqueda en tiempo real de productos.
   - Detalle de Producto dinámico cargado por ID (`detalle.html?id=X`).
   - Sistema de Comentarios vinculado a usuarios reales.
   - Panel Administrativo Completo (CRUD) protegido por Login.
   - Registro e Inicio de Sesión de Clientes Finales.
   - Inyección asíncrona de Imágenes HD desde CDNs oficiales (Hotlinking).
   - Footer personalizado con firma del desarrollador colegiado.

3. ESTRUCTURA DE ARCHIVOS CREADA
   /frontend/
   ├── index.html         (Página principal / Catálogo)
   ├── detalle.html       (Vista individual de producto)
   ├── admin.html         (Panel Privado CRUD)
   ├── login.html         (Acceso Administrador)
   ├── login_cliente.html (Acceso Público)
   ├── styles.css         (Diseño global)
   └── scripts.js         (Lógica principal de UI y consumo de API)
   
   /backend/
   ├── app.py             (Servidor Flask Principal)
   ├── init_db.py         (Script creador de la Base de Datos Relacional)
   ├── inyectar_imagenes.py (Script optimizador de Imágenes CDN)
   ├── update_db.py       (Script de semilla de descripciones)
   └── database.db        (Archivo físico de SQLite)

4. ¿CÓMO EJECUTAR EL PROYECTO EN EL FUTURO?
   Paso 1: Abrir la terminal en la carpeta raíz del proyecto.
   Paso 2: Ejecutar el backend (servidor de datos) 
           Comando: `python backend/app.py`
           (O `python3 backend/app.py` dependiendo del SO).
   Paso 3: Una vez que la consola diga "Running on http://127.0.0.1:5000",
           puedes abrir directamente los archivos HTML en tu navegador.
           El punto de entrada principal es `frontend/index.html`.

5. NOTAS DEL DESARROLLADOR
   - La base de datos incluye la tabla 'roles' con (1=Admin, 2=Cliente).
   - Las validaciones de seguridad se realizan tanto del lado del cliente 
     (evitando peticiones nulas) como del lado del servidor.
   - Se cumplió al 100% la rúbrica de Requerimientos 1 y 2.
